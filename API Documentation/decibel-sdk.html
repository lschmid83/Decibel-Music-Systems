Welcome to the Decibel SDK

About

The Decibel SDK makes it simple to query the Decibel API and deal with the results returned.

Calls to the Decibel API are made through query objects, which take care of all the logic involved in running a REST query, including deserialising the results to handy result objects. These can then be used to easily ingegrate Decibel data into your own application

Installation

  • In Visual Studio, right click the References section of your project in the Solution Explorer and select Add Reference
  • Click Browse, navigate to the location you have saved the DLL to and confirm the addition by clicking OK
  • Add the following line to the top of any file that will use the Decibel C# SDK
using DecibelSDK;

Querying The API

Back to top

Before you can query the Decibel API, you must first set your app ID and app Key. These are provided on registration.

DecibelUtilities.SetKeys("YOUR_APP_ID", "YOUR_APP_KEY");

Each query topic has its own Query class. A full list of topics can be found in our API Reference. SingleQuery classes, eg SingleAlbumQuery, return one result and are used to find items by ID. SearchQuery classes, eg AlbumsQuery, return many results and are used to search the API by parameters such as title, main artists, release date etc.

Querying works the same for both Single and Search topics. First, a new instance of the class must be created. The search parameters should then be filled in with the values you require. The Decibel C# SDK is IntelliSense compatible. A full list of parameters can also be found online in our API Reference. Please note: not all search fields are currently supported by the SDK

The following code demonstrates how to create a new AlbumSearch query, looking for albums with the title 'Rumours' that were released in 1977.

var qo = new AlbumsQuery
{
    Title = "Rumours",
    DateReleased = "1977",
    Depth = new AlbumRetrievalDepth?[] { AlbumRetrievalDepth.ArtistDetails, AlbumRetrievalDepth.Genres },
};

The following code demonstrates how to create a new SingleAlbum query to find the album with a given Spotify ID.

var qo = new SingleAlbumQuery()
{
    Id = "spotify:album:{SPOTIFY_ID}",
    IdType = AlbumSearchType.Spotify,
};

Once the parameters you wish to search on have been set, use the Run() method to run the query.

qo.Run();

Dealing With Results

Back to top

Before doing anything with the results of the query, it is good practice to check if any errors occurred.

// Check if any errors occurred
if (qo.Error.HasError)
{
    // Error.ErrorMsg provides details of the error
    Console.WriteLine(qo.Error.ErrorMsg);
}

The result of the query is saved to the Result object in your Query object. The structure of this differs slightly for Single and Search queries.

Single Query Results

SingleQuery topics return a copy of the terms of the query in a Search object, (see below) and a Result object. The type of this object corresponds to the Query topic, so a SingleAlbumQuery would return an Album. The properties of this object can then be accessed and used within your own application.

// Get the album requested from the result
var album = qo.Result;
 
// Display the details of the album
Console.WriteLine("Title: " + album.Title);
Console.WriteLine("Artist: " + album.ArtistsLiteral);
Console.WriteLine("ReleaseDate: " + album.OriginalReleaseDate);

Search Query Result

All Search queries return a Result array, as oppose to a single Result. The type of this again corresponds to the topic, so a AlbumsQuery would return a Result[] of Album[].

var albums = qo.Result;
foreach (var album in albums)
{
    // Display the details of the album
    Console.WriteLine("Title: " + album.Title);
    Console.WriteLine("Artist: " + album.ArtistsLiteral);
    Console.WriteLine("ReleaseDate: " + album.OriginalReleaseDate);
}

The Result[] can be manipulated using LINQ statements to search within and order results.

// Only get albums in the ResultSet by Fleetwood Mac
var fleetwoodMac = qo.Result.Where(x => x.ArtistsLiteral == "Fleetwood Mac");
// Order the results by title
var alphabetical = qo.Result.OrderBy(x => x.Title);

In addition to the ResultSet and a copy of the terms of the query, Search queries return a PageCount, ResultCount and TotalResultCount.

Query Terms

The terms of your original query are returned in the Search object.

var searchTerms = qo.Search;

Planned Changes
Back to top

Below is a list of planned changes for future releases that will likely effect the way you use the Decibel C# SDK. This list is not comprehensive and is subject to change at any time whilst the SDK is in beta release.

  • The underlying object model is currently being re-written, which may result in changes to some Query and Result objects. Expected v0.2 and upwards
  • The qo.Search object that contains details of your search terms will be removed once the paging utilities have been released. Expected v0.4 - v0.5
