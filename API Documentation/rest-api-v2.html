Introduction

Thanks for taking a look at our API. On this page you can read about how to get the most out of it. To get access, you’ll need to register with us – you can do that here. The first three months are on us!

Our API is structured to make our detailed metadata as easy for you to access as possible. Working across a number of topics, you can either search for key terms or specify one of our IDs. To tell us what information you want, include a depth parameter too – all this is outlined in the reference key below. Happy coding!

Overview

Data Structure

The Decibel API represents the clearest way to capture all the relevant data about music and recordings. This means there are a couple of things that might take you by surprise: firstly, an album is different from a release – different releases of the same album may have different track listings, dates, etc. – and secondly, tracks are not the same as recordings – the same recording may appear on different releases, meaning that it’s a different “track” on each.

So we do things a little differently at Decibel. To keep things clean, we stick to albums and stay away from particular album releases, unless you ask us not to. And instead of dealing with tracks, we do recordings. Read the below, and have a look around the API – it won’t take long to get used to.

Searching by ID

You can search the Decibel API by various external IDs, including IDs for Spotify and Discogs, as well as by Decibel’s own IDs – you just need to specify which ID type you are using. All the available ID types are listed in the documentation reference below.

Topics

Searching

Our URLs firstly need a sub-URL to specify what type of item you are looking for. If you’re searching, this is one of the following topics:

  • Albums
  • Artists
  • Disctags
  • Recordings

By ID

You can look up items by a range of IDs. We have our own Decibel IDs, but Decibel also works with Spotify IDs, Discogs IDs, and various others too – see below to find out more. The topics you can use with these IDs are:

  • Album
  • Artist
  • Image
  • Recording

So a basic query will look like this:

http://rest.decibel.net/v2/artists?title=fleetwood%20mac

or

http://rest.decibel.net/v2/artist?id=4aea91de-318f-e311-be87-ac220b82800d

Retrieval Depth

By default, the API will return a basic object with no extra information. All further information can be requested by specifying a retrieval depth. This means that we only provide the information you ask for, and we don’t waste bandwidth and time giving you details you don’t want. You can specify multiple retrieval depths, separating them with a semicolon. Valid depth parameters for each possible sub-URL are detailed in the reference below.

Return Values

Objects are returned as JSON by default, or alternatively XML. The type of object returned depends on the topic – for a full list of topics and the objects they return, see the reference below.

Deserializing Results

We provide XSD schemas for every return type: click here to download them. These can be used to deserialize the results of an API request. There are tools avalible for most languages and development envoirnments that will create an object model based off the schemas for this purpose. For your convience, we have provided object models for C# and Java.

Take a look at the Sample code section below for more resources.

Paging

When Decibel performs a query, it returns all of the IDs found. However, it only resolves the first page of items. To request another page, you need to specify this using the parameter pageNumber=. The first page number is 1.

Images

Images are currently only available for albums, though in the future we will have them for participants, performances and other items too. Standard images are 600x600 pixels, and thumbnails are 95x95. You do need to specify what type of image you need, either using /standard or /thumbnail, or by using the parameter ?imageType=standard or ?imageType=thumbnail.


REST API REFERENCE

album

Back to top
Finds an album by Id

Request URL /album
Verb GET
Returns SingleAlbumQueryResult
Parameters
Name Required Description
depth N Information to bring back
Valid values are:
  • Genres - Genres of the recording
  • Identifiers - Catalog numbers, bar codes, media checksums, etc.
  • None - Minimum information
  • Participations - Track and album Participations
  • Recordings - Include track listings
  • Works - Works on each recording - includes authors if participants is also set
id Y Valid album ID
idType N Type of ID - from list
Valid values are:
  • CatalogNumber - Label catalog number
  • Decibel - Decibel ID: generic version of an album
  • EAN - 13-digit bar code
  • FreeDBChecksum - Disk checksum in FreeDB database
  • FullName - Names exactly matching this string
  • PartialName - Names containing this string
  • Spotify - Spotify locator
Examples
http://rest.decibel.net/v2/album?id=156dda6c-358f-e311-be87-ac220b82800d&depth=Recordings;Identifiers;Works;Participations;Genres;ArtistDetails;Releases;

Verb GET

artist N The name of the main or featured artist

  • EAN - 13-digit bar code
